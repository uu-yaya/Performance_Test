# 桌宠性能采集操作指南：从环境准备开始

本文档用于实际执行测试。按顺序做，不要跳步。

## 0. 测试前约定

默认路径：

```powershell
$ProjectDir = "D:\desktop_pet"
$ExcelPath = "D:\desktop_pet\桌宠类产品性能测试记录表.xlsx"
$Collector = "D:\desktop_pet\tools\collect_desktop_pet_perf.ps1"
$Importer = "D:\desktop_pet\tools\import_perf_results.py"
$Python = "C:\Users\zoeuuliu\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe"
```

如果测试 BongoCat，默认：

```powershell
$ProductId = "P001"
$ProductName = "BongoCat"
$ProcessName = "BongoCat"
```

如果进程名不确定，先启动产品，然后运行：

```powershell
Get-Process | Sort-Object ProcessName | Select-Object ProcessName,Id,CPU,WorkingSet64 | Out-GridView
```

看列表里产品对应的进程名。`BongoCat.exe` 在脚本里写 `BongoCat` 或 `BongoCat.exe` 都可以。

## 1. 重启电脑

先手动重启电脑。

重启后不要立刻测试，等待 5 分钟，让 Windows 更新、杀毒、索引、云同步等后台任务稳定。

可以用命令记录当前时间：

```powershell
Get-Date
```

---

## 2. 关闭干扰应用

建议关闭：

- 浏览器
- 微信/QQ/Discord
- Steam 下载或更新
- 云盘同步
- 游戏启动器
- 录屏/直播软件
- IDE
- 其他桌宠、壁纸软件

查看当前高占用进程：

```powershell
Get-Process |
  Sort-Object CPU -Descending |
  Select-Object -First 20 ProcessName,Id,CPU,@{Name="MemoryMB";Expression={[math]::Round($_.WorkingSet64/1MB,1)}} |
  Format-Table -AutoSize
```

查看内存占用最高的进程：

```powershell
Get-Process |
  Sort-Object WorkingSet64 -Descending |
  Select-Object -First 20 ProcessName,Id,@{Name="MemoryMB";Expression={[math]::Round($_.WorkingSet64/1MB,1)}} |
  Format-Table -AutoSize
```

如果看到下载器、浏览器、录屏软件等明显干扰项，手动关闭。

## 3. 固定系统状态

记录系统版本：

```powershell
Get-ComputerInfo |
  Select-Object WindowsProductName,WindowsVersion,OsHardwareAbstractionLayer,OsTotalVisibleMemorySize |
  Format-List
```

记录 CPU：

```powershell
Get-CimInstance Win32_Processor |
  Select-Object Name,NumberOfCores,NumberOfLogicalProcessors,MaxClockSpeed |
  Format-List
```

记录 GPU：

```powershell
Get-CimInstance Win32_VideoController |
  Select-Object Name,AdapterRAM,DriverVersion,VideoModeDescription |
  Format-List
```

记录显示器分辨率：

```powershell
Get-CimInstance Win32_VideoController |
  Select-Object Name,CurrentHorizontalResolution,CurrentVerticalResolution,CurrentRefreshRate |
  Format-Table -AutoSize
```

记录电源模式：

```powershell
powercfg /GETACTIVESCHEME
```

记录电池/接电状态，笔记本很重要：

```powershell
Get-CimInstance Win32_Battery |
  Select-Object BatteryStatus,EstimatedChargeRemaining |
  Format-List
```

没有输出通常表示台式机或系统未暴露电池信息。

## 5. 采集系统基线

这一步不要启动桌宠。

运行 5 分钟，用于判断当前桌面环境是否稳定：

```powershell
powershell -ExecutionPolicy Bypass -File D:\desktop_pet\tools\collect_desktop_pet_perf.ps1 `
  -ProductId BASELINE `
  -ProductName SystemBaseline `
  -ProcessName explorer `
  -Scenario 系统基线 `
  -DurationSec 300 `
  -IntervalSec 1
```

说明：

- 这不是全系统所有进程的采集，而是采 explorer 作为桌面环境参考。
- 如果这一步 CPU/GPU 都异常高，先不要测产品，继续排查后台干扰。

---

## 6. 启动待测桌宠

手动启动 BongoCat 或其他待测产品。

启动后等待 30-60 秒，让产品进入稳定状态。

确认进程存在：

```powershell
Get-Process -Name BongoCat -ErrorAction SilentlyContinue |
  Select-Object ProcessName,Id,CPU,@{Name="MemoryMB";Expression={[math]::Round($_.WorkingSet64/1MB,1)}} |
  Format-Table -AutoSize
```

如果没有输出，说明进程名不对。用这条命令找：

```powershell
Get-Process | Sort-Object ProcessName | Select-Object ProcessName,Id | Out-GridView
```

## 7. 采集启动峰值

如果要测启动峰值，应该先关闭产品，然后运行命令后立刻启动产品。

命令：

```powershell
powershell -ExecutionPolicy Bypass -File D:\desktop_pet\tools\collect_desktop_pet_perf.ps1 `
  -ProductId P001 `
  -ProductName BongoCat `
  -ProcessName BongoCat `
  -Scenario 启动峰值 `
  -DurationSec 120 `
  -IntervalSec 1
```

执行方式：

1. 先关闭 BongoCat。
2. 运行上面命令。
3. 命令开始后立刻启动 BongoCat。
4. 等待 2 分钟采集结束。

## 10. 采集功能开启场景

如果产品有 AI、语音、麦克风、摄像头、高帧率、Live2D、本地模型等功能，开启后采 10 分钟。

## 11. 采集长时间稳定

运行 1 小时：

```powershell
powershell -ExecutionPolicy Bypass -File D:\desktop_pet\tools\collect_desktop_pet_perf.ps1 `
  -ProductId P001 `
  -ProductName BongoCat `
  -ProcessName BongoCat `
  -Scenario 长时间稳定 `
  -DurationSec 3600 `
  -IntervalSec 1
```

如果要更严格，可以跑 4 小时：

```powershell
powershell -ExecutionPolicy Bypass -File D:\desktop_pet\tools\collect_desktop_pet_perf.ps1 `
  -ProductId P001 `
  -ProductName BongoCat `
  -ProcessName BongoCat `
  -Scenario 长时间稳定4小时 `
  -DurationSec 14400 `
  -IntervalSec 1
```

重点看：

- 内存是否持续增长
- 句柄是否持续增长
- 线程是否持续增长
- CPU 是否从低占用慢慢漂移到高占用

## 12. 游戏影响测试

这一步用于评估桌宠对游戏 FPS、1% Low、帧时间的影响。

### 12.1 先采游戏单独运行

1. 关闭桌宠。
2. 打开游戏，进入固定场景。
3. 用 PresentMon 或 NVIDIA FrameView 采集 10 分钟。
4. 保存 CSV 到：

```text
D:\desktop_pet\external_logs\presentmon_game_only.csv
```

### 12.2 再采游戏 + 桌宠

1. 启动桌宠。
2. 打开同一个游戏、同一个场景、同一套画质设置。
3. 用 PresentMon 或 NVIDIA FrameView 采集 10 分钟。
4. 保存 CSV 到：

```text
D:\desktop_pet\external_logs\presentmon_game_with_pet.csv
```

同时运行桌宠采集：

```powershell
powershell -ExecutionPolicy Bypass -File D:\desktop_pet\tools\collect_desktop_pet_perf.ps1 `
  -ProductId P001 `
  -ProductName BongoCat `
  -ProcessName BongoCat `
  -Scenario 游戏影响 `
  -DurationSec 600 `
  -IntervalSec 1
```
